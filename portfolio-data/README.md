# Portfolio Data Layer

This folder contains **machine-readable JSON files** automatically generated from the daily learning markdown files in this repository.

They are designed to be consumed directly by a portfolio website — no markdown parsing required.

---

## Files

| File | Description |
|---|---|
| `timeline.json` | Full chronological list of all learning entries |
| `latest.json` | The single most-recent entry |
| `stats.json` | Aggregate statistics across all entries |
| `tags.json` | Tag → frequency map, sorted by most used |
| `featured.json` | Top 10 entries ranked by depth and complexity |
| `knowledge-base.json` | AI-assistant-ready entries with full concept extraction |

---

## Data Shape

### `timeline.json`
```jsonc
[
  {
    "id": "day-035",               // sequential ID, zero-padded
    "date": "2026-06-08",          // ISO 8601 date
    "title": "VaultZero — Day 3",  // primary topic of the day
    "summary": "...",              // 1–3 sentence description
    "tags": ["Encryption", "TypeScript"],
    "difficulty": "Advanced",      // Beginner | Intermediate | Advanced
    "sourceFile": "08-06-2026.md",
    "githubUrl": "https://github.com/...",
    "wordCount": 1345
  }
]
```

### `latest.json`
```jsonc
{
  "id": "day-035",
  "date": "2026-06-08",
  "title": "VaultZero — Day 3",
  "summary": "...",
  "tags": [...],
  "difficulty": "Advanced",
  "githubUrl": "https://github.com/...",
  "generatedAt": "2026-06-08T18:29:13.506Z"
}
```

### `stats.json`
```jsonc
{
  "totalLearningDays": 35,
  "totalTopics": 141,
  "totalTags": 40,
  "totalTechnologies": 23,
  "totalWordsWritten": 21430,
  "difficultyBreakdown": { "Beginner": 14, "Intermediate": 17, "Advanced": 4 },
  "firstEntry": "2026-04-30",
  "lastUpdated": "2026-06-08",
  "generatedAt": "..."
}
```

### `tags.json`
```jsonc
{
  "Salesforce": 35,
  "Encryption": 24,
  "Security": 19,
  "Networking": 15,
  ...
}
```

### `featured.json`
Array of the 10 most complex/in-depth entries (ranked by difficulty then word count). Same shape as a `timeline.json` entry, with the addition of a `technologies` field.

### `knowledge-base.json`
```jsonc
{
  "generatedAt": "...",
  "totalEntries": 35,
  "entries": [
    {
      "id": "day-035",
      "date": "2026-06-08",
      "title": "...",
      "summary": "...",
      "concepts": ["Implemented encryptVault / decryptVault", "..."],
      "technologies": ["TypeScript", "AES-256-GCM", "BIP-39"],
      "topics": ["vault/types.ts — The Data Model", "..."],
      "tags": ["Encryption", "TypeScript"],
      "difficulty": "Advanced",
      "links": [{ "label": "VaultZero GitHub", "url": "https://..." }],
      "source": "https://github.com/..."
    }
  ]
}
```

## Portfolio Website Integration

Fetch any file directly from the raw GitHub URL:

```js
// Fetch latest entry
const latest = await fetch(
  'https://raw.githubusercontent.com/Sanskar-bot/Daily-Learnings/main/portfolio-data/latest.json'
).then(r => r.json());

// Fetch full timeline
const timeline = await fetch(
  'https://raw.githubusercontent.com/Sanskar-bot/Daily-Learnings/main/portfolio-data/timeline.json'
).then(r => r.json());

// Fetch stats
const stats = await fetch(
  'https://raw.githubusercontent.com/Sanskar-bot/Daily-Learnings/main/portfolio-data/stats.json'
).then(r => r.json());
```

> **Tip:** GitHub raw URLs are cached. For a portfolio you control, you can also serve them from GitHub Pages or any static CDN.

## Regenerating Locally

```bash
node scripts/generate-portfolio-data.js
```

Or via npm:

```bash
npm run generate
```

No dependencies required — uses only Node.js built-ins (`fs`, `path`). Works on Node 18+.

---

## Automation

The JSON files are auto-regenerated by the GitHub Action at `.github/workflows/update-portfolio-data.yml`.

**Trigger:** Every push to `main` that modifies a `*.md` file or the generator script itself.

**What it does:**
1. Runs `node scripts/generate-portfolio-data.js`
2. Stages only the `portfolio-data/` directory
3. Commits with message `chore(portfolio-data): auto-regenerate JSON [skip ci]` and pushes

The `[skip ci]` tag prevents the commit from triggering a second workflow run.

---

## Adding a New Learning Day

Your workflow is **unchanged**. Just create a new markdown file anywhere in the `2026/` folder following the same `DD-MM-YYYY.md` naming convention, push to `main`, and the JSON files update automatically within ~30 seconds.
