#### `vault/types.ts` — The Data Model

Before writing any vault logic, the type definitions had to be airtight. TypeScript's `strict: true` mode enforces these — no implicit `any`, no unchecked nulls.

```typescript
// vault/types.ts

export interface VaultEntry {
  readonly id:        string;      // UUID v4 — client-generated, never server-assigned
  readonly site:      string;      // "github.com" — domain name, not URL
  readonly username:  string;
  readonly password:  string;      // NEVER persisted in plaintext
  readonly label?:    string;      // user-assigned display name (optional)
  readonly notes?:    string;      // free text, also encrypted
  readonly createdAt: string;      // ISO 8601 — client clock, not server
  readonly updatedAt: string;
}

// The plaintext vault — lives only in service worker memory
export interface Vault {
  readonly version:   number;       // schema version — for future migration
  readonly entries:   VaultEntry[];
}

// What the server stores — opaque to the server
export interface EncryptedVault {
  readonly ciphertext: string;     // base64(IV + AES-GCM ciphertext + authTag)
  readonly updatedAt:  string;     // ISO 8601 — for last-write-wins conflict resolution
}
```

The `version` field in `Vault` is important: if the entry schema changes in a future release (e.g., adding a `totp` field), the decrypted vault's version number tells the app which migration function to run before presenting entries to the user.

#### `vault/vault.ts` — Encrypt and Decrypt the Full Vault

The vault is a JSON object. To encrypt it:
1. Serialize to JSON string
2. Encode to UTF-8 bytes
3. AES-256-GCM encrypt with the vault key
4. Base64-encode the resulting blob for transmission

```typescript
// vault/vault.ts
import { encryptAESGCM, decryptAESGCM } from '../crypto/aes-gcm.js';
import { toBase64, fromBase64, toUTF8Bytes, fromUTF8Bytes } from '../utils/encoding.js';

export async function encryptVault(
  vault: Vault,
  vaultKey: Uint8Array
): Promise<EncryptedVault> {
  try {
    const plaintext = toUTF8Bytes(JSON.stringify(vault));
    const encrypted = await encryptAESGCM(plaintext, vaultKey);
    return {
      ciphertext: toBase64(encrypted),
      updatedAt:  new Date().toISOString()
    };
  } catch (err) {
    throw new Error(`Vault encryption failed: ${String(err)}`);
  }
}

export async function decryptVault(
  encrypted: EncryptedVault,
  vaultKey: Uint8Array
): Promise<Vault> {
  try {
    const blob      = fromBase64(encrypted.ciphertext);
    const plaintext = await decryptAESGCM(blob, vaultKey);
    const json      = fromUTF8Bytes(plaintext);
    const vault     = JSON.parse(json) as Vault;
    // Validate version before returning
    if (typeof vault.version !== 'number' || !Array.isArray(vault.entries)) {
      throw new Error('Vault schema validation failed');
    }
    return vault;
  } catch (err) {
    throw new Error(`Vault decryption failed: ${String(err)}`);
  }
}
```

The `JSON.parse` cast to `Vault` is intentionally followed by a runtime validation check — TypeScript's type assertions are compile-time only. If the decrypted blob is corrupted or from an incompatible version, the validation catches it before the entries are used.

#### `recovery/bip39.ts` — 24-Word Recovery Phrase

A recovery phrase gives users a human-readable backup of their vault key. If the master password is forgotten, the recovery phrase can re-derive access — but only if the user has stored it safely.

**How BIP-39 works:**
1. Generate 256 bits of entropy from CSPRNG
2. Append a checksum: SHA-256(entropy), take the first 8 bits
3. Split 264 bits into 24 groups of 11 bits each
4. Map each 11-bit number to one of 2048 English words from the BIP-39 wordlist

```typescript
// recovery/bip39.ts
export async function generateRecoveryPhrase(): Promise<string> {
  const entropy  = crypto.getRandomValues(new Uint8Array(32)); // 256 bits
  const hash     = await crypto.subtle.digest('SHA-256', entropy);
  const checksum = new Uint8Array(hash)[0]; // first 8 bits of SHA-256(entropy)

  const bits    = new Uint8Array([...entropy, checksum]);
  const indices = extractBIP39Indices(bits);   // 24 × 11-bit indices

  return indices.map(i => WORDLIST[i]).join(' ');
}

export function validateRecoveryPhrase(phrase: string): boolean {
  const words = phrase.trim().toLowerCase().split(/\s+/);
  if (words.length !== 24) return false;
  return words.every(w => WORDLIST.includes(w));
}
```

The recovery phrase is shown to the user exactly once during registration. The app never stores it — only a SHA-256 hash of the phrase is stored in the database (`User.recoveryHash`). This means the server can verify "this phrase is valid" without ever being able to reconstruct it.

#### `utils/encoding.ts` — The Boring but Essential Layer

Every crypto function produces `Uint8Array`. The database stores strings. The API sends JSON. These helpers bridge the gap without any ambiguity:

```typescript
// utils/encoding.ts

/** Uint8Array → base64 string */
export const toBase64 = (bytes: Uint8Array): string =>
  Buffer.from(bytes).toString('base64');

/** base64 string → Uint8Array */
export const fromBase64 = (b64: string): Uint8Array =>
  new Uint8Array(Buffer.from(b64, 'base64'));

/** Uint8Array → lowercase hex string */
export const toHex = (bytes: Uint8Array): string =>
  Buffer.from(bytes).toString('hex');

/** lowercase hex string → Uint8Array */
export const fromHex = (hex: string): Uint8Array =>
  new Uint8Array(Buffer.from(hex, 'hex'));

/** string → UTF-8 Uint8Array */
export const toUTF8Bytes = (str: string): Uint8Array =>
  new TextEncoder().encode(str);

/** UTF-8 Uint8Array → string */
export const fromUTF8Bytes = (bytes: Uint8Array): string =>
  new TextDecoder().decode(bytes);

/** Constant-time comparison — prevents timing attacks on hash comparisons */
export function constantTimeEqual(a: Uint8Array, b: Uint8Array): boolean {
  if (a.length !== b.length) return false;
  let diff = 0;
  for (let i = 0; i < a.length; i++) {
    diff |= a[i]! ^ b[i]!;      // XOR — accumulates any bit differences
  }
  return diff === 0;              // only true if no bits differed
}
```

`constantTimeEqual` is worth explaining: a naive `===` comparison of two strings short-circuits as soon as it finds a mismatch. An attacker who can measure how long comparisons take can use this to guess characters one by one — a **timing attack**. `constantTimeEqual` always iterates the full length regardless of where the mismatch is, making execution time constant and timing attacks impossible.

#### `@vaultzero/core` — The Complete Public API

All modules are now re-exported through clean barrel files:

```typescript
// core/src/index.ts
export * from './crypto/index.js';    // deriveKEK, encryptAESGCM, decryptAESGCM, ...
export * from './vault/index.js';     // encryptVault, decryptVault, VaultEntry, ...
export * from './recovery/index.js';  // generateRecoveryPhrase, validateRecoveryPhrase
export * from './utils/index.js';     // toBase64, fromBase64, toHex, constantTimeEqual, ...
```

Any other package (`@vaultzero/backend`, the extension, the web app) can now import:
```typescript
import { deriveKEK, encryptVault, generateRecoveryPhrase } from '@vaultzero/core';
```
