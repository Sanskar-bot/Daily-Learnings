#### Why TanStack Start over Next.js / Vite SPA?

The first decision was the framework. Three options were on the table:

| Framework | SSR | File-based Routing | React 19 | Bundle Size | Verdict |
|-----------|-----|--------------------|----------|-------------|---------|
| Next.js 15 | ✅ | ✅ | ✅ | Large | Heavy — too opinionated |
| Vite SPA | ❌ | ❌ | ✅ | Minimal | No SSR, manual routing |
| **TanStack Start** | ✅ | ✅ | ✅ | Lean | **✅ Chosen** |

TanStack Start is the full-stack mode of TanStack Router — it provides file-based routing, optional SSR, and works with Vite 7 as the bundler. For a portfolio that needs good Lighthouse/SEO scores while remaining a modern React 19 app, this is the right call. The routing is typed end-to-end: route params and search params are inferred at the type level, eliminating an entire class of runtime errors.

#### Tailwind CSS v4 — The OKLCH Shift

Tailwind v4 dropped the `tailwind.config.js` model. Design tokens now live in CSS using `@theme` directives, and all colors must be expressed in **OKLCH** (Oklab Lightness Chroma Hue).

Why OKLCH over hex/HSL?

```
oklch(L C H)
         ↑ ↑ ↑
         │ │ └─ Hue angle (0–360°)
         │ └─── Chroma (colorfulness, 0–0.4 typical)
         └───── Lightness (0 = black, 1 = white, perceptually uniform)
```

HSL lightness is *not* perceptually uniform — `hsl(120 100% 50%)` green looks far brighter than `hsl(240 100% 50%)` blue at the same `L` value. OKLCH is perceptually uniform: two colors at the same `L` value look equally bright to the human eye. This matters when building a palette where cyan and magenta both serve as accents — they need to feel equally prominent, not compete.

The design tokens defined in `styles.css`:

```css
@theme {
  /* Core palette */
  --background:   oklch(0.08 0.02 260);   /* Deep navy-black */
  --foreground:   oklch(0.96 0.01 240);   /* Near-white text */
  --primary:      oklch(0.82 0.18 170);   /* Neon cyan-green */
  --accent:       oklch(0.70 0.22 320);   /* Neon magenta */
  --muted:        oklch(0.20 0.03 260);   /* Subtle surface */
  --border:       oklch(0.25 0.05 260);   /* Faint border */

  /* Typography */
  --font-sans:    'Space Grotesk', system-ui, sans-serif;
  --font-mono:    'JetBrains Mono', 'Fira Code', monospace;
}
```

#### Custom Utility Classes — The Reusable Layer

Rather than applying raw Tailwind classes everywhere, a set of semantic utility classes were defined that encapsulate the design language:

```css
/* Text gradient — cyan → magenta, 135° */
.text-gradient {
  background: linear-gradient(135deg, var(--primary), var(--accent));
  -webkit-background-clip: text;
  -webkit-text-fill-color: transparent;
  background-clip: text;
}

/* Glassmorphism panel */
.glass {
  background: oklch(0.12 0.03 260 / 0.6);
  backdrop-filter: blur(12px) saturate(1.5);
  border: 1px solid oklch(0.30 0.05 260 / 0.4);
}

/* Neon glowing border */
.neon-border {
  border: 1px solid var(--primary);
  box-shadow:
    0 0 8px  oklch(0.82 0.18 170 / 0.4),
    0 0 16px oklch(0.82 0.18 170 / 0.2);
}

/* Dot grid background */
.grid-bg {
  background-image: radial-gradient(
    oklch(0.82 0.18 170 / 0.12) 1px,
    transparent 1px
  );
  background-size: 28px 28px;
}
```

The `glass` class is critical to the aesthetic: every info card and panel uses glassmorphism instead of solid backgrounds. The `backdrop-filter: blur(12px)` requires the element to have a semi-transparent background — `oklch(... / 0.6)` is the alpha syntax.

#### The `__root.tsx` Layout — Global Providers

TanStack Start's root layout wraps every page. The setup:

```typescript
// app/routes/__root.tsx
export default function RootComponent() {
  return (
    <html lang="en">
      <head>
        <meta charset="UTF-8" />
        <meta name="viewport" content="width=device-width, initial-scale=1.0" />
        <title>Sanskar Phougat — Cybersecurity Researcher & Security Engineer</title>
        <meta name="description" content="Portfolio of Sanskar Phougat — cybersecurity researcher, security engineer, and B.Tech student at JIIT. Specializing in applied cryptography, offensive security, and zero-knowledge protocols." />
        <meta property="og:title" content="Sanskar Phougat — Cybersecurity Portfolio" />
        <meta property="og:type" content="website" />
        <link rel="preconnect" href="https://fonts.googleapis.com" />
        <link href="https://fonts.googleapis.com/css2?family=Space+Grotesk:wght@400;500;600;700&family=JetBrains+Mono:wght@400;500&display=swap" rel="stylesheet" />
      </head>
      <body>
        <CustomCursor />
        <Outlet />
      </body>
    </html>
  );
}
```

The `<CustomCursor />` is injected here — outside any section component — so it overlays the entire page regardless of scroll position.

#### Navbar — Scroll-Aware Glass Effect

The Navbar uses a React `useEffect` with a `scroll` event listener to toggle a CSS class when the user scrolls past 30px:

```typescript
// components/Navbar.tsx
useEffect(() => {
  const handleScroll = () => setScrolled(window.scrollY > 30);
  window.addEventListener('scroll', handleScroll, { passive: true });
  return () => window.removeEventListener('scroll', handleScroll);
}, []);
```

The `{ passive: true }` option is important — it tells the browser this listener will never call `preventDefault()`, allowing the browser to scroll without waiting for JavaScript. Without it, Chrome's compositor thread stalls on every scroll event.

At rest: `background: transparent`, no blur.
After 30px scroll: `backdrop-filter: blur(12px)` + `background: oklch(0.08 0.02 260 / 0.8)`.

The transition is `transition: all 0.3s ease` — smooth enough to feel polished, fast enough not to lag behind the user's scroll momentum.

#### Custom Cursor — Zero-Lag via `requestAnimationFrame`

The `CustomCursor` component bypasses React's rendering pipeline entirely:

```typescript
// components/CustomCursor.tsx
useEffect(() => {
  const dot = document.createElement('div');
  dot.style.cssText = `
    position: fixed;
    width: 8px; height: 8px;
    border-radius: 50%;
    background: white;
    pointer-events: none;
    mix-blend-mode: difference;
    z-index: 9999;
    transform: translate(-50%, -50%);
  `;
  document.body.appendChild(dot);

  let x = 0, y = 0;
  const onMove = (e: MouseEvent) => { x = e.clientX; y = e.clientY; };
  window.addEventListener('mousemove', onMove);

  let raf: number;
  const tick = () => {
    dot.style.left = x + 'px';
    dot.style.top  = y + 'px';
    raf = requestAnimationFrame(tick);
  };
  raf = requestAnimationFrame(tick);

  return () => {
    window.removeEventListener('mousemove', onMove);
    cancelAnimationFrame(raf);
    document.body.removeChild(dot);
  };
}, []);
```

The `mix-blend-mode: difference` makes the cursor invert whatever color is under it — white on dark backgrounds, dark on light backgrounds. This works without any JavaScript color computation: the GPU handles it entirely via compositing. The `requestAnimationFrame` loop drives position updates at the display's native refresh rate (60/120 Hz), synchronized to vsync — smoother than any CSS transition could achieve.
