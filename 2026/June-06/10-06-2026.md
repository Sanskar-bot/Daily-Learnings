#### HeroScene — Custom WebGL Globe with Three.js

`HeroScene.tsx` is the most technically complex component. Built entirely with `@react-three/fiber` and `@react-three/drei` — no prebuilt globe library. The full architecture:

**1. The Globe Mesh — IcosahedronGeometry Wireframe**

```typescript
// HeroScene.tsx
<mesh ref={globeRef}>
  <icosahedronGeometry args={[2, 4]} />   {/* radius=2, detail=4 */}
  <meshBasicMaterial
    color={primaryColor}
    wireframe
    transparent
    opacity={0.15}
  />
</mesh>
```

`IcosahedronGeometry` at detail level 4 produces 1280 triangular faces — enough resolution to look like a globe wireframe without being geometrically expensive. `wireframe: true` renders only the edges, not the faces, so the 3D lines form the characteristic globe grid pattern.

**2. City Nodes — Lat/Lon to 3D Cartesian**

Real-world city coordinates are converted to points on the sphere surface using the spherical coordinate transform:

```typescript
function latLonToVec3(lat: number, lon: number, radius: number): THREE.Vector3 {
  const phi   = (90 - lat)  * (Math.PI / 180);  // polar angle from +Y axis
  const theta = (lon + 180) * (Math.PI / 180);  // azimuthal angle from -Z axis

  return new THREE.Vector3(
    -radius * Math.sin(phi) * Math.cos(theta),  // X
     radius * Math.cos(phi),                    // Y (up)
    radius * Math.sin(phi) * Math.sin(theta)   // Z
  );
}
```

25 cities are defined with their actual latitude/longitude — Tokyo, New York, London, São Paulo, Mumbai, Sydney, etc. The coordinate transform places each city exactly where it should be on the sphere surface.

Node rendering uses `InstancedMesh` — a single draw call renders all 25 city dots:

```typescript
const nodeGeometry = new THREE.SphereGeometry(0.04, 8, 8);
const nodeMaterial = new THREE.MeshBasicMaterial({ color: primaryColor });
// InstancedMesh: 1 draw call for all 25 nodes
const instancedMesh = new THREE.InstancedMesh(nodeGeometry, nodeMaterial, nodes.length);
```

Without instancing, 25 separate sphere meshes would generate 25 draw calls — the GPU overhead multiplies. One `InstancedMesh` with 25 instances is a single draw call. For 25 nodes it's negligible, but it establishes the right pattern for when node count scales.

**3. Connection Arcs — Quadratic Bézier Curves**

For each city, the 3 nearest cities are found (Euclidean distance on the sphere surface), and a Bézier arc connects them:

```typescript
function createArc(start: THREE.Vector3, end: THREE.Vector3): THREE.BufferGeometry {
  // Control point: midpoint of the chord, pushed outward by 30%
  const mid = new THREE.Vector3().lerpVectors(start, end, 0.5);
  mid.multiplyScalar(1.3);   // push outward — creates the arc's bow

  const curve = new THREE.QuadraticBezierCurve3(start, mid, end);
  const points = curve.getPoints(50);    // 50 segments → smooth curve
  return new THREE.BufferGeometry().setFromPoints(points);
}
```

The control point (`mid.multiplyScalar(1.3)`) is the key: it pushes the midpoint of the chord 30% outward from the sphere centre. This gives the arc its characteristic bow shape — a great-circle approximation that looks geographically correct.

Arc colors use vertex coloring for the cyan-to-transparent gradient effect:

```typescript
const colors = new Float32Array(points.length * 3);
for (let i = 0; i < points.length; i++) {
  const t = i / (points.length - 1);
  // Fade: full cyan at endpoints, slightly dimmer at midpoint
  colors[i * 3 + 0] = 0.0;    // R
  colors[i * 3 + 1] = 0.9 - t * 0.2;  // G — slight fade
  colors[i * 3 + 2] = 0.9 - t * 0.1;  // B
}
geometry.setAttribute('color', new THREE.BufferAttribute(colors, 3));
```

**4. Data Packets — Animated Points on Arcs**

40 data packets travel along the arcs in real-time. Each packet is a small sphere that follows a Bézier curve at its own speed and phase:

```typescript
// In useFrame() — called every render frame (~60fps)
packets.forEach((packet, i) => {
  packet.t += packet.speed;    // advance along arc (0 → 1)
  if (packet.t > 1) packet.t = 0;   // loop back to start

  const pos = packet.curve.getPoint(packet.t);
  // Update instanced mesh position for this packet
  dummy.position.set(pos.x, pos.y, pos.z);
  dummy.updateMatrix();
  packetMesh.setMatrixAt(i, dummy.matrix);
});
packetMesh.instanceMatrix.needsUpdate = true;
```

The `dummy` is a reusable `THREE.Object3D` — it's transformed then its matrix extracted and written into the instanced mesh. This avoids allocating a new matrix object every frame. Over 60 frames/second × 40 packets, this saves 2400 object allocations per second.

**5. Mouse Reactivity — Cursor Speed Modulation**

The globe reacts to cursor velocity, not just position. Mouse speed is calculated each frame:

```typescript
const mouseDelta = useRef({ x: 0, y: 0, speed: 0 });

useEffect(() => {
  let prevX = 0, prevY = 0;
  const onMove = (e: MouseEvent) => {
    const dx = e.clientX - prevX;
    const dy = e.clientY - prevY;
    mouseDelta.current.speed = Math.sqrt(dx * dx + dy * dy);
    prevX = e.clientX; prevY = e.clientY;
  };
  window.addEventListener('mousemove', onMove);
  return () => window.removeEventListener('mousemove', onMove);
}, []);

// In useFrame():
const speed = mouseDelta.current.speed;
// Arc opacity pulses with cursor speed
arcMaterial.opacity = 0.3 + Math.min(speed * 0.01, 0.4);
// Globe tilts slightly with cursor X position
globeRef.current.rotation.y += 0.002 + speed * 0.0001;
```

This creates a subtle feedback loop: moving the cursor faster makes the globe pulse and spin slightly faster. The arc opacity modulation (`0.3 + ...`) keeps arcs always visible at rest but energised by fast cursor movement.

#### About Section — GSAP ScrollTrigger Stagger

The About section uses GSAP's ScrollTrigger to stagger-reveal each paragraph as the user scrolls:

```typescript
// About.tsx
useEffect(() => {
  const ctx = gsap.context(() => {
    gsap.from('.about-line', {
      opacity: 0,
      y: 30,
      duration: 0.8,
      stagger: 0.15,   // each line starts 150ms after the previous
      ease: 'power2.out',
      scrollTrigger: {
        trigger: '.about-section',
        start: 'top 75%',
        toggleActions: 'play none none reverse',
      }
    });
  });
  return () => ctx.revert();  // cleanup on unmount
}, []);
```

The `gsap.context()` wrapper is critical — it scopes all animations to the component and ensures `ctx.revert()` cleans up all GSAP animations and ScrollTrigger instances when the component unmounts. Without this, scroll listeners and timelines accumulate across hot-module-replacement cycles in development, causing subtle animation bugs.
