#### Why Light Mode Is Non-Trivial for a Dark-First Site

Most "light mode" implementations are an afterthought вАФ just swap `background: black` for `background: white` and call it done. That approach breaks entirely on a site built around OKLCH neon accents (cyan `oklch(0.82 0.18 170)`, magenta `oklch(0.70 0.22 320)`) because:

1. **Neon colors on white backgrounds look garish** вАФ neon cyan on white has insufficient contrast and feels harsh rather than premium.
2. **Glassmorphism breaks** вАФ `.glass` uses `backdrop-filter: blur(12px)` with a dark semi-transparent fill. On a white background, the same values look like foggy grey smudges.
3. **The dot-grid background** (`radial-gradient` with cyan dots at 12% opacity) disappears entirely on a white background.
4. **`mix-blend-mode: difference`** on the custom cursor вАФ which inverts underlying colors вАФ produces a black cursor dot on white backgrounds, which is fine, but the dot becomes invisible over dark text.

The solution is a dedicated light-mode design token set вАФ not a color inversion, but a carefully considered second palette.

#### The Light Mode OKLCH Palette

The OKLCH color model was already in use from Tailwind v4's `@theme` directive. The new light-mode tokens:

```css
/* Light mode overrides вАФ applied via [data-theme="light"] on <html> */
[data-theme="light"] {
  --background:   oklch(0.97 0.01 240);   /* Warm off-white вАФ not pure #fff */
  --foreground:   oklch(0.15 0.02 260);   /* Near-black with a blue undertone */
  --primary:      oklch(0.45 0.20 200);   /* Teal-blue вАФ readable on light bg */
  --accent:       oklch(0.45 0.22 310);   /* Deep magenta/violet */
  --muted:        oklch(0.90 0.02 240);   /* Subtle surface вАФ slightly cooler than bg */
  --border:       oklch(0.78 0.04 240);   /* Visible but not harsh */
}
```

Key decisions:
- **`oklch(0.97 ...)` not `oklch(1.0 ...)`** вАФ pure white (`L=1.0`) feels clinical and flat. `L=0.97` with a tiny chroma gives the background warmth without a visible color tint.
- **`--primary` shifted from `L=0.82` to `L=0.45`** вАФ in dark mode, high-lightness primary (`0.82`) pops on the dark background. On light, it would be near-invisible. `L=0.45` gives the same hue but dark enough for WCAG AA contrast (4.5:1 minimum against `oklch(0.97 ...)`).
- **Chroma kept high** (`0.20вАУ0.22`) вАФ chroma is colorfulness. Dropping it would make the palette feel dull and desaturated. The light mode should feel equally vibrant, just in a different register.

#### CSS Custom Properties вАФ Theme Switching Architecture

The theme toggle works by setting a `data-theme` attribute on the `<html>` element. All components use `var(--background)`, `var(--foreground)`, etc. вАФ so the cascade handles the switch automatically:

```css
/* styles.css */
:root {
  /* Dark mode defaults */
  --background:   oklch(0.08 0.02 260);
  --foreground:   oklch(0.96 0.01 240);
  --primary:      oklch(0.82 0.18 170);
  --accent:       oklch(0.70 0.22 320);
  --muted:        oklch(0.20 0.03 260);
  --border:       oklch(0.25 0.05 260);
}

[data-theme="light"] {
  --background:   oklch(0.97 0.01 240);
  --foreground:   oklch(0.15 0.02 260);
  --primary:      oklch(0.45 0.20 200);
  --accent:       oklch(0.45 0.22 310);
  --muted:        oklch(0.90 0.02 240);
  --border:       oklch(0.78 0.04 240);
}
```

The transition between themes is animated вАФ `transition: background-color 0.3s ease, color 0.3s ease` applied globally so the switch feels smooth rather than jarring.

#### Glassmorphism in Light Mode

The `.glass` utility needed a light-mode variant:

```css
.glass {
  background: oklch(0.12 0.03 260 / 0.6);
  backdrop-filter: blur(12px) saturate(1.5);
  border: 1px solid oklch(0.30 0.05 260 / 0.4);
}

[data-theme="light"] .glass {
  background: oklch(0.97 0.01 240 / 0.75);   /* Near-white, semi-transparent */
  backdrop-filter: blur(16px) saturate(1.8);  /* More blur for frosted effect */
  border: 1px solid oklch(0.78 0.04 240 / 0.6);
}
```

Increasing `blur` from `12px` to `16px` in light mode compensates for the reduced contrast of a light blur вАФ the extra blur makes the "frosted glass" metaphor hold visually.

#### The Theme Toggle Button вАФ Design

The toggle button sits in the Navbar, right of the nav links. It cycles between `вШАпЄП Light` and `рЯМЩ Dark` with a smooth icon swap. The icon uses `opacity` + `transform` transitions so the swap feels like a physical flip:

```typescript
// ThemeToggle.tsx
const [theme, setTheme] = useState<'dark' | 'light'>('dark');

const toggle = () => {
  const next = theme === 'dark' ? 'light' : 'dark';
  document.documentElement.setAttribute('data-theme', next);
  localStorage.setItem('theme', next);
  setTheme(next);
};
```

`localStorage` persistence means the user's preference survives page refreshes. On initial load, a small inline script (before React hydrates) reads the stored preference and sets `data-theme` on `<html>` immediately вАФ preventing a flash of the wrong theme (FOUT).
