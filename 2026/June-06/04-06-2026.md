## What I Learned

### VaultZero — Zero-Knowledge Password Manager (Project Planning)

> **Context:** Starting a new personal project — VaultZero.
> A cross-platform password manager that stores passwords, generates
> strong passwords, works as a browser extension, and autofills
> credentials on Android and iOS. The core principle: zero-knowledge
> encryption — the server never sees your plaintext passwords.

#### The Core Idea

Most password managers work like a bank locker where the bank also has a copy of your key. If the bank gets robbed, your locker gets opened too.

VaultZero works differently.

Passwords are scrambled (encrypted) on the user's own device before they ever leave it. The server only receives the scrambled version — like a locked box it has no key for. Even if someone breaks into the server and steals everything, all they get is a pile of locked boxes they cannot open.

The only thing that can unscramble the passwords is the user's **master password**, which **never leaves the device**.

#### What This Means in Practice

| Threat Scenario | What Happens |
|---|---|
| Server is hacked | Attacker gets nothing readable — only encrypted blobs |
| Employee goes rogue | They see only scrambled data — no plaintext access |
| Database is stolen | Useless without the user's master password |
| Fake website (phishing) | The app warns the user before filling credentials |

#### Honest Limitation

If malware is already running on the user's own device while the vault is open, no password manager in the world can defend against that. This is an honest limitation of every client-side solution, not a design flaw.

**In short:** your passwords are yours. The server is just a storage box that cannot read what it is storing.

#### Planned Features

| Feature | Description |
|---|---|
| Password Storage | Encrypted vault stored locally + synced to server as encrypted blobs |
| Password Generator | Configurable generator — length, symbols, passphrases, entropy display |
| Browser Extension | Chrome/Firefox extension (Manifest V3) — detect login forms, autofill, save new credentials |
| Android App | Native or React Native app with biometric unlock + autofill service integration |
| iOS App | Native or React Native app with AutoFill Credential Provider Extension |
| Cross-Device Sync | Encrypted vault synced across all platforms via the server — server never decrypts |
| Phishing Detection | URL matching before autofill — warns if the domain doesn't match the saved entry |

#### Architecture Philosophy

- **Client-side encryption** — AES-256-GCM, keys derived from master password via Argon2id
- **Zero-knowledge server** — stores only encrypted blobs + metadata (timestamps, device IDs)
- **End-to-end encrypted sync** — vault changes are encrypted before upload, decrypted after download
- **No password recovery** — if the master password is lost, the vault is lost (by design, not by accident)
- **Offline-first** — the vault works without internet; sync happens when connectivity is available

#### Target Platforms

1. **Browser Extension** — Chrome & Firefox (Manifest V3, esbuild bundling, React popup UI)
2. **Android** — React Native or native Kotlin, with Android Autofill Framework integration
3. **iOS** — React Native or native Swift, with AutoFill Credential Provider Extension
4. **Backend API** — Express.js + Prisma + PostgreSQL (stores encrypted vault blobs, handles auth)
5. **Shared Core** — TypeScript crypto library (`@vaultzero/core`) shared across all clients
