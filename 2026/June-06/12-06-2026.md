#### Step 1 — CSS Token Override in `styles.css`

Wrote the `[data-theme="light"]` block directly after the `:root` block in `styles.css`. All component styles already used `var(--background)`, `var(--foreground)`, etc., so no component files needed color changes — the cascade did the work automatically:

```css
/* styles.css — light mode token overrides */
[data-theme="light"] {
  --background:   oklch(0.97 0.01 240);
  --foreground:   oklch(0.15 0.02 260);
  --primary:      oklch(0.45 0.20 200);
  --accent:       oklch(0.45 0.22 310);
  --muted:        oklch(0.90 0.02 240);
  --border:       oklch(0.78 0.04 240);
}

/* Smooth theme transition — all elements */
*, *::before, *::after {
  transition: background-color 0.3s ease, color 0.3s ease,
              border-color 0.3s ease, box-shadow 0.3s ease;
}
```

The `*` selector for transitions is broad but necessary — without it, individual elements would pop between colors instead of fading. The `0.3s ease` duration is fast enough to feel instant but slow enough to be perceivable as intentional.

#### Step 2 — Light Mode Glassmorphism & Dot Grid

The `.glass` utility and `.grid-bg` both needed explicit light-mode overrides since they use hardcoded `oklch()` values, not CSS variables:

```css
[data-theme="light"] .glass {
  background: oklch(0.97 0.01 240 / 0.75);
  backdrop-filter: blur(16px) saturate(1.8);
  border: 1px solid oklch(0.78 0.04 240 / 0.6);
}

[data-theme="light"] .neon-border {
  border-color: var(--primary);
  box-shadow:
    0 0 6px  oklch(0.45 0.20 200 / 0.35),
    0 0 12px oklch(0.45 0.20 200 / 0.18);
}

[data-theme="light"] .grid-bg {
  background-image: radial-gradient(
    oklch(0.45 0.20 200 / 0.10) 1px,
    transparent 1px
  );
  background-size: 28px 28px;
}
```

The `.neon-border` shadow colors are reduced in opacity (`0.35` vs `0.4`, `0.18` vs `0.2`) — light backgrounds make glows look more pronounced, so slightly less opacity achieves the same perceptual intensity.

The `.grid-bg` dot color switches from the neon cyan at 12% opacity to the teal-blue primary at 10% opacity — still visible as a subtle texture but not harsh.

#### Step 3 — `ThemeToggle.tsx` Component

```typescript
// components/ThemeToggle.tsx
import { useState, useEffect } from 'react';
import { SunIcon, MoonIcon } from '@heroicons/react/24/outline';

export function ThemeToggle() {
  const [theme, setTheme] = useState<'dark' | 'light'>('dark');

  // Read persisted preference on mount
  useEffect(() => {
    const stored = localStorage.getItem('theme') as 'dark' | 'light' | null;
    if (stored) {
      setTheme(stored);
      document.documentElement.setAttribute('data-theme', stored);
    }
  }, []);

  const toggle = () => {
    const next = theme === 'dark' ? 'light' : 'dark';
    document.documentElement.setAttribute('data-theme', next);
    localStorage.setItem('theme', next);
    setTheme(next);
  };

  return (
    <button
      onClick={toggle}
      aria-label={`Switch to ${theme === 'dark' ? 'light' : 'dark'} mode`}
      className="p-2 rounded-lg glass neon-border transition-all duration-300 hover:scale-110"
    >
      {theme === 'dark' ? (
        <SunIcon className="w-5 h-5 text-primary" />
      ) : (
        <MoonIcon className="w-5 h-5 text-primary" />
      )}
    </button>
  );
}
```

Key implementation details:
- **`useEffect` for localStorage read** — `localStorage` is not available during SSR (TanStack Start runs on Node.js). Reading inside `useEffect` guarantees it only runs in the browser after hydration.
- **`aria-label`** — dynamically describes the *next* action (not the current state), which is the correct ARIA pattern for toggle buttons.
- **`hover:scale-110`** — subtle scale-up on hover reinforces interactivity without needing a separate animation.

#### Step 4 — FOUT Prevention (Flash of Unstyled Theme)

Without this step, users with a saved light-mode preference would see the dark theme flash for ~100ms before React hydrates and applies the stored preference. The fix: an inline `<script>` in `<head>`, before any CSS:

```typescript
// __root.tsx — in <head>, before stylesheets
<script
  dangerouslySetInnerHTML={{
    __html: `
      (function() {
        var t = localStorage.getItem('theme');
        if (t === 'light') {
          document.documentElement.setAttribute('data-theme', 'light');
        }
      })();
    `,
  }}
/>
```

This runs synchronously before the browser paints anything — `data-theme="light"` is set on `<html>` before any CSS is applied, so the correct theme tokens are used from the very first paint. No flash.

#### Step 5 — WebGL Globe Theme Reactivity

The Three.js materials in `HeroScene.tsx` use hardcoded color strings — they don't read CSS variables. A `useTheme` hook reads the current `data-theme` attribute and passes the appropriate color to the globe:

```typescript
// hooks/useTheme.ts
import { useState, useEffect } from 'react';

export function useTheme() {
  const [theme, setTheme] = useState<'dark' | 'light'>('dark');

  useEffect(() => {
    const observer = new MutationObserver(() => {
      const t = document.documentElement.getAttribute('data-theme');
      setTheme(t === 'light' ? 'light' : 'dark');
    });
    observer.observe(document.documentElement, { attributes: true, attributeFilter: ['data-theme'] });
    return () => observer.disconnect();
  }, []);

  return theme;
}
```

`MutationObserver` watches the `<html>` element for changes to `data-theme` — when the toggle fires, the observer callback runs synchronously, updating React state. `HeroScene.tsx` then uses:

```typescript
const theme = useTheme();
const primaryColor = theme === 'light' ? '#1a7a8c' : '#5eebc4';  // teal vs cyan
```

The globe smoothly transitions its neon accent color to the light-mode teal when the theme switches — no page reload required.
