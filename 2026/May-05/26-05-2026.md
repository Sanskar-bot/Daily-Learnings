#### Attack Types — Comprehensive Research

**Malware**
- Umbrella term for malicious software designed to damage, disrupt, or gain unauthorized access
- **Viruses** — attach to legitimate files and execute when the host file is opened; require human action to spread
- **Worms** — self-replicate across networks without human interaction; exploit network vulnerabilities (e.g., WannaCry exploited EternalBlue/SMB vulnerability)
- **Trojans** — disguise as legitimate software; once installed, create backdoors for attackers
- **Ransomware** — encrypts victim's files, demands payment (usually cryptocurrency) for the decryption key
- **Spyware** — covert surveillance software that monitors keystrokes, browsing habits, and credentials
- Key distinction: viruses need a host file; worms propagate independently across networks

**Phishing**
- Social engineering attack using fraudulent emails, websites, or messages to trick users into revealing credentials or installing malware
- **Spear Phishing** — targeted at specific individuals using personal information to increase credibility
- **Whaling** — spear phishing targeting C-suite executives and senior leadership
- **Vishing** — voice phishing via phone calls impersonating banks, IT support, or government agencies
- **Smishing** — SMS-based phishing with malicious links or urgent messages
- Still the **#1 initial attack vector globally** — responsible for over 90% of successful breaches (Verizon DBIR)

**DoS / DDoS**
- **Denial of Service (DoS)** — overwhelming a target with traffic from a single source to exhaust resources
- **Distributed DoS (DDoS)** — same attack from thousands of compromised machines (botnet)
- Three types:
  - **Volumetric** — bandwidth flooding (UDP flood, DNS amplification)
  - **Protocol** — exploiting protocol weaknesses (SYN flood, Ping of Death)
  - **Application Layer** — targeting specific services (HTTP flood, Slowloris)
- Mitigation: rate limiting, CDN/DDoS protection services (Cloudflare, AWS Shield), traffic analysis

**Man-in-the-Middle (MITM)**
- Attacker secretly intercepts and potentially alters communication between two parties who believe they are communicating directly
- Common scenarios: unsecured public Wi-Fi, ARP spoofing, DNS hijacking, SSL stripping
- The attacker can eavesdrop (passive) or modify messages in transit (active)
- Mitigated by TLS/SSL, certificate pinning, end-to-end encryption, and HSTS

**SQL Injection**
- Inserting malicious SQL queries through user input fields to manipulate backend databases
- Can lead to data exfiltration, authentication bypass, or complete database destruction
- Example: entering `' OR '1'='1` in a login field to bypass authentication
- Prevention: parameterized queries (prepared statements), input validation, WAFs, least-privilege database accounts

**Password Attacks**
- **Brute force** — trying every possible combination systematically
- **Dictionary attack** — using a precompiled list of common passwords and word combinations
- **Credential stuffing** — using leaked username/password pairs from other breaches (exploits password reuse)
- **Rainbow table attack** — using precomputed hash tables to reverse password hashes
- Mitigation: MFA, account lockout policies, password complexity requirements, salted hashing

**Social Engineering**
- Manipulating human psychology rather than exploiting technical vulnerabilities
- **Pretexting** — creating a fabricated scenario to extract information (e.g., impersonating IT support)
- **Baiting** — offering something enticing (infected USB drives, free downloads) to lure victims
- **Tailgating** — physically following an authorized person into a restricted area
- **Quid pro quo** — offering a service in exchange for information (e.g., "free tech support" that installs malware)
- The weakest link in any security system is often the human element
