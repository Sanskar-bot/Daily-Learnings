## What I Learned

### E2E Encryption Attack Lab — Act 1: Plaintext Interception

> **Project Context:** Building a 3-act MITM demonstration for the CyberPeace Foundation assignment.
> Act 1 establishes the baseline — showing how unencrypted TCP traffic is trivially intercepted.

#### Project Structure & Setup
- Designed the full folder structure: `dashboard/`, `screenshots/`, `report/`
- Installed core dependencies: `cryptography`, `mitmproxy`, `flask`
- Understood the role of each dependency in the overall system

#### `crypto_utils.py` — Cryptographic Helpers
- Implemented **RSA-2048** key generation, encryption, and decryption
- Implemented **AES-256-GCM** authenticated encryption (encrypt + decrypt)
- Implemented **SHA-256 fingerprinting** of public keys — the foundation for Act 3's verification mechanism
- Learned why GCM mode is preferred: it provides both confidentiality *and* integrity via authentication tags

#### `server.py` — TCP Relay
- Built a dumb TCP relay on **port 9000** that simply forwards bytes between two connected clients
- Understood the architectural decision: keeping the server "dumb" makes it a realistic stand-in for any intermediary (ISP, cloud relay, etc.)
- Learned how raw TCP sockets work — `socket.accept()`, threading per connection, `recv()`/`sendall()` loops

#### `alice.py` & `bob.py` — Plain Mode
- Built both clients in plain (unencrypted) mode first — sends and receives raw UTF-8 messages
- This establishes the Act 1 baseline before encryption is added in later acts

#### Burp Suite — Proxy Configuration
- Configured Burp Suite as a proxy intercepting on **port 8080**
- Set an upstream proxy rule to forward traffic to **port 9000** (the relay)
- Learned about Burp's upstream proxy feature and how it chains proxies

#### Wireshark — Loopback Traffic Capture
- Captured Act 1 evidence on the loopback interface (`lo` / `127.0.0.1`)
- Verified plaintext messages visible in the **ASCII pane** — proving zero confidentiality with no encryption
- Understood why loopback capture is useful for local protocol demos

#### Challenge Solved — Burp Upstream Proxy Not Forwarding
- **Problem:** Messages were being dropped — Burp wasn't forwarding raw TCP to port 9000 correctly.
- **Root cause:** When a proxy host was set in the upstream rule, Burp was attempting to re-proxy the connection as HTTP rather than forwarding raw TCP.
- **Fix:** Left the upstream proxy host blank and set destination to `127.0.0.1:9000` — this told Burp to forward the raw TCP stream directly without re-interpretation.
