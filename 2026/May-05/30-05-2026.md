#### OSI Model — All 7 Layers Explained

The **Open Systems Interconnection (OSI) Model** is a conceptual framework that standardizes network communication into 7 distinct layers, each with a specific responsibility.

**Layer 7 — Application Layer**
- The interface between the user/application and the network
- Protocols: **HTTP, HTTPS, DNS, FTP**
- This is where the user directly interacts — web browsers, email clients, file transfer tools
- DNS resolves domain names to IP addresses; HTTP/HTTPS handles web communication

**Layer 6 — Presentation Layer**
- Responsible for data formatting, encryption, and compression
- Protocols: **TLS, SSL**
- Translates data between the application and network format — ensures both ends interpret data the same way
- Handles character encoding (ASCII, Unicode), image formats (JPEG, PNG), and encryption/decryption

**Layer 5 — Session Layer**
- Manages sessions (connections) between applications
- Protocols: **NetBIOS, RPC**
- Establishes, maintains, and terminates communication sessions
- Handles session checkpointing and recovery — if a transfer is interrupted, it can resume from the last checkpoint

**Layer 4 — Transport Layer**
- End-to-end delivery of data with error control and flow control
- Protocols: **TCP, UDP**
- **TCP** — connection-oriented, reliable delivery, three-way handshake (SYN → SYN-ACK → ACK), guarantees order and delivery
- **UDP** — connectionless, fast but unreliable, no handshake, used for streaming, gaming, DNS queries
- Port numbers live here — they identify specific applications/services on a host

**Layer 3 — Network Layer**
- Logical addressing and routing of packets across networks
- Protocols: **IP, ICMP**
- **IP addresses** — unique identifiers for devices across the internet (IPv4: 32-bit, IPv6: 128-bit)
- **Routing** — determining the best path for packets to travel from source to destination
- **ICMP** — used for diagnostics (ping, traceroute) and error reporting

**Layer 2 — Data Link Layer**
- Physical addressing and frame delivery within a local network
- Protocols: **Ethernet, ARP**
- **MAC addresses** — unique hardware identifiers burned into network interface cards
- **ARP (Address Resolution Protocol)** — maps IP addresses to MAC addresses within a local network
- Handles frame encapsulation, error detection (CRC), and media access control

**Layer 1 — Physical Layer**
- Raw bit transmission over physical media
- Technologies: **Wi-Fi, Fiber Optic, Ethernet cables, Bluetooth**
- Deals with voltage levels, data rates, physical connectors, pin layouts
- Defines the electrical, optical, or radio specifications for data transmission

#### Key Protocol Deep Dives

**HTTP (Hypertext Transfer Protocol)**
- Foundation of web communication — client sends requests, server sends responses
- Stateless protocol — each request is independent
- Methods: GET (retrieve), POST (submit), PUT (update), DELETE (remove)

**TCP vs UDP — When to Use Which**

| Feature | TCP | UDP |
|---|---|---|
| Connection | Connection-oriented (3-way handshake) | Connectionless |
| Reliability | Guaranteed delivery, ordered | Best-effort, no order guarantee |
| Speed | Slower (overhead for reliability) | Faster (minimal overhead) |
| Use Cases | Web browsing, email, file transfer | Video streaming, gaming, DNS |
| Header Size | 20 bytes minimum | 8 bytes |

**DNS (Domain Name System)**
- Translates human-readable domain names (google.com) to IP addresses (142.250.x.x)
- Hierarchical system: Root → TLD (.com, .org) → Authoritative DNS
- Uses UDP port 53 for queries (TCP for zone transfers)

**ARP (Address Resolution Protocol)**
- Resolves IP addresses to MAC addresses within a local network segment
- ARP Request: "Who has IP 192.168.1.1?" — broadcast to all devices on the LAN
- ARP Reply: "I have 192.168.1.1, my MAC is AA:BB:CC:DD:EE:FF" — unicast response

#### Network Architecture & Data Flow
- Data flows **down** the OSI stack at the sender (encapsulation) and **up** at the receiver (decapsulation)
- Each layer adds its own header (and sometimes trailer) — this is called **encapsulation**
- At Layer 4: data becomes **segments** (TCP) or **datagrams** (UDP)
- At Layer 3: segments become **packets** (with IP headers)
- At Layer 2: packets become **frames** (with MAC addresses)
- At Layer 1: frames become **bits** transmitted over the physical medium
