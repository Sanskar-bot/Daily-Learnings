#### Attack Lifecycle — Revised & Expanded (Post-Feedback)

Based on mentor feedback, restructured the attack lifecycle into a clear 3-phase model with detailed sub-steps:

**Phase 1 — Surveillance / Reconnaissance**
- Attacker gathers information about the target before launching any attack
- **Passive reconnaissance:** OSINT (Open Source Intelligence) — social media profiling, public records, job postings that reveal technology stack, WHOIS lookups
- **Active reconnaissance:** network scanning (Nmap), port scanning, vulnerability scanning (Nessus, OpenVAS)
- **Social reconnaissance:** identifying key personnel, organizational structure, email formats (firstname.lastname@org.com)
- Goal: build a complete picture of the target's attack surface — people, systems, and processes

**Phase 2 — Exploitation**
- Using gathered intelligence to breach the target system
- **Phishing / Spear Phishing** — the most common initial access vector (90%+ of breaches)
- **Vulnerability exploitation** — leveraging known CVEs or zero-day vulnerabilities in unpatched software
- **Credential attacks** — brute force, credential stuffing, or using harvested credentials from reconnaissance
- **Watering hole attacks** — compromising a website the target is known to visit
- Goal: gain initial access — a foothold inside the target network

**Phase 3 — Access & Persistence**
- Establishing long-term, covert control of the compromised environment
- **Backdoors** — installing persistent access mechanisms that survive reboots and updates
- **Privilege escalation** — moving from a low-privilege user account to domain admin (e.g., using Mimikatz)
- **Lateral movement** — spreading across the network to reach high-value targets (file servers, databases, backup systems)
- **Data exfiltration** — stealing data before deploying ransomware or destructive payloads
- **Covering tracks** — clearing logs, timestomping, living-off-the-land to avoid detection
- Goal: achieve the attacker's objective — data theft, ransomware deployment, or long-term espionage

#### Connecting Attack Types to the Lifecycle
- Phishing & social engineering → **Phase 1 & 2** (reconnaissance + initial access)
- Malware (trojans, ransomware) → **Phase 2 & 3** (exploitation + persistence)
- SQL Injection → **Phase 2** (exploitation of application vulnerabilities)
- MITM attacks → **Phase 2 & 3** (interception during active operations)
- DoS/DDoS → Can be **Phase 2** (disruption) or a distraction during data exfiltration
